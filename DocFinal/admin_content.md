# 管理者端：內容管理模組

---

## 6.1 靜態頁面管理

管理前台資訊頁的文字內容，無需改程式碼即可更新。

| 頁面 | slug | 說明 |
|------|------|------|
| 尺寸指南 | size_guide | 畫布尺寸對照與場景建議 |
| 出貨流程 | shipping | 訂單出貨步驟說明 |
| 訂製流程 | custom_process | 客製化服務完整流程說明 |
| 報價參考 | pricing_reference | 尺寸 × 難易度價格區間表 + 說明 |
| 退款退貨政策 | refund_policy | 退款條件與流程說明 |

**操作：** 選擇頁面 → 編輯標題與內容（Markdown）→ 儲存

## 6.2 系統設定

儲存全站共用的可設定參數。

| 設定項目 | key | 說明 |
|---------|-----|------|
| 銀行帳號 | bank_account_number | 結帳頁顯示的匯款帳號 |
| 銀行名稱 | bank_name | 匯款銀行名稱 |
| 戶名 | bank_account_name | 匯款戶名 |
| 客製申請預計回覆天數 | quote_reply_days | 提交後顯示「X 個工作天內回覆」|
| 畫具內容物說明 | product_info_tools | 商品頁折疊區塊內容（Markdown）|
| 畫布材質說明 | product_info_material | 同上 |
| 使用建議 | product_info_tips | 同上 |
| 注意事項 | product_info_notes | 同上 |
| 顏料塗佈係數 | paint_ml_per_cm2 | 每 cm² 所需顏料 ml 數（需實測後填入，預設 0.05）|
| 顏料最低配給量 | paint_min_ml | 每色最低配給 ml 數（預設 3）|
| 顏料緩衝係數 | paint_buffer_ratio | 安全緩衝倍數（預設 1.3，即多 30%）|
| 客製服務費率 | custom_photo_price_multiplier | 客製照片售價相對於目錄公式的倍數（預設 2.0）|
| 付款絕對期限（小時）| payment_absolute_deadline_hours | 訂單從建立時起算的付款最終期限，防止反覆 flag 長期佔用庫存（預設 48）|

## 6.2.1 畫布尺寸管理

獨立於系統設定，以表格介面管理 `canvas_sizes`。

| 操作 | 說明 |
|------|------|
| 新增尺寸 | 填寫寬、高、顯示名稱、排序 |
| 編輯尺寸 | 修改顯示名稱、排序；**不可修改寬高**（會破壞既有 production_jobs / orders 的語意）|
| 停用尺寸 | 設 is_active=false，該尺寸不在新商品/客製表單中出現，但不影響已建立的 production_jobs 和 orders |
| 查看使用情況 | 顯示每個尺寸有幾筆 production_jobs、幾筆上架 variants |

> 新增尺寸時**需同步更新** pricing_formula 的基礎成本表（印製費 + 框架費 + 固定費），否則該尺寸的商品無法計算定價。這部分由管理員手動維護。

## 6.3 客製服務報價設定

獨立頁面，管理客製照片的基礎報價表與加費項目。

**基礎定價表（`custom_photo_prices`）**

管理員設定各「尺寸 × 難易度」的基礎報價，供報價流程自動帶入：

| 操作 | 說明 |
|------|------|
| 查看定價表 | 以尺寸 × 難易度矩陣顯示所有基礎價格 |
| 編輯金額 | 點選儲存格直接修改金額 |

**加費項目（`custom_photo_surcharges`）**

管理員設定照片類型、人物數量、複雜度等加費項目：

| 操作 | 說明 |
|------|------|
| 查看加費列表 | 顯示所有加費項目、金額、啟用狀態 |
| 新增加費項目 | 填寫類別、名稱、金額 |
| 編輯金額 | 修改現有加費金額 |
| 啟用 / 停用 | 控制報價時是否顯示此項目 |

## 6.4 客製完成案例管理

管理前台「客製化商品」頁展示的完成案例。

| 操作 | 說明 |
|------|------|
| 新增案例 | 上傳成品圖、填寫標題、說明、類型、規格 |
| 編輯案例 | 修改任意欄位 |
| 刪除案例 | 從前台移除 |
| 上架 / 下架 | 控制是否公開顯示（`is_published`）|
| 管理類型標籤 | 新增 / 編輯 / 刪除案例分類（如：人像、寵物、風景、建築）；刪除分類時該分類下的案例 `category_id` 自動設為 null，前台顯示為「未分類」|

## 6.5 銷售報表

獨立頁面，初期顯示基本統計，後續視需求擴充。

| 區塊 | 說明 |
|------|------|
| 日期區間篩選 | 選擇起訖日期（預設：本月）|
| 總訂單數 | 期間內建立的訂單筆數 |
| 總營收 | 期間內 `status=completed` 且未退款（`refund_amount` 為 null 或 0）的訂單 `total` 加總；部分退款訂單以 `total - refund_amount` 計入 |

> 後續可擴充：商品銷售排行、折扣使用統計、退款率等。

## 6.6 資料庫欄位

**案例類型標籤（case_categories）**

| 欄位 | 說明 |
|------|------|
| id | 主鍵 |
| name | 類型名稱（如「人像」、「寵物」）|
| created_at | 建立時間 |

> `custom_cases.category_id` FK → case_categories.id

---

**靜態頁面（static_pages）**

| 欄位 | 說明 |
|------|------|
| id | 主鍵 |
| slug | 頁面識別碼（唯一）|
| title | 頁面標題 |
| content | 頁面內容（Markdown）|
| updated_at | 最後更新時間 |

**系統設定（system_settings）**

| 欄位 | 說明 |
|------|------|
| key | 設定項目識別碼（唯一）|
| value | 設定值 |
| updated_at | 最後更新時間 |
