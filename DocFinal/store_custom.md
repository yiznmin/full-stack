# 用戶商店端：客製化商品頁

---

## 頁面結構

```
客製化商品
  ├── 完成案例展示（上方）
  ├── 客製照片申請（自帶照片製作專屬套件）
  └── 客製規格申請（現有圖片，想要不同尺寸/難易度/細緻度）
```

> 客製規格也可從商品詳情頁直接進入，此頁提供獨立入口給未從商品頁進來的用戶。

---

## 完成案例展示

- 管理員後台上傳的客製完成案例（圖片 + 簡短說明）
- 卡片式展示，顯示：成品圖、類型標籤（由管理員後台自訂）、尺寸、難易度
- 點選卡片 → 展開詳情（大圖、完整說明）
- 詳情頁底部有「諮詢類似規格」按鈕 → 點選後預填表單（帶入該案例的尺寸、難易度偏好）

---

## 客製照片申請表單

此表單為 `custom_photo` 類型（客戶自帶照片，管理員製作專屬套件）。

| 欄位 | 必填 | 說明 |
|------|------|------|
| 服務類型 | — | 固定為「客製照片」|
| 上傳照片 | ✓ | JPG / PNG，檔案大小上限 10MB，超過時前端顯示錯誤 |
| 畫布尺寸偏好 | 選填 | 下拉選單（從 `canvas_sizes` 表讀取 is_active=true 的尺寸），未填時由管理員根據照片建議 |
| 難易度偏好 | ✓ | 入門 / 初級 / 中級 / 進階 / **讓管理員建議**（存 null）|
| 備注 / 特殊需求 | 選填 | 自由文字 |

> **移除細緻度欄位**：客製照片的細緻度由管理員根據照片品質決定（符合 pricing_formula.md「細緻度由管理員根據照片品質決定，不影響定價」），客戶無須填寫。

- 需登入才能提交
- **未登入時**：訪客可填寫表單，點「送出」時：
  1. 顯示提示：「資料已暫存，請登入後自動送出，無需重新填寫」
  2. 表單資料存入 `sessionStorage`（key：`pending_custom_form`）
  3. 跳轉登入頁（帶 `redirect` 參數）
  4. 登入成功後導回，前端自動讀取 `sessionStorage` 送出申請，並清除暫存
- 提交後顯示「申請已送出，管理員將於 X 個工作天內回覆報價」（X 從 system_settings 讀取）
- 狀態可在「會員中心 → 客製申請」查看

---

## 客製規格申請

> **客製規格（custom_spec）不提供獨立入口**，僅能從「商品詳情頁 > 想要不同規格？」按鈕進入，此時 ref_product_id 自動帶入。此規則確保 custom_spec 一定關聯到既有商品。

---

## 案例管理（後台）

管理員可在後台：
- 新增案例（上傳成品圖、填寫說明、類型標籤、規格）
- 編輯 / 刪除案例
- 設定案例是否公開顯示

**案例資料表（custom_cases）**

| 欄位 | 說明 |
|------|------|
| id | 主鍵 |
| image_url | 成品圖（Firebase Storage）|
| title | 案例標題 |
| description | 說明文字 |
| category_id | FK → case_categories.id（管理員自訂類型）|
| canvas_w_cm | 尺寸寬 |
| canvas_h_cm | 尺寸高 |
| difficulty | 難易度 |
| is_published | 是否公開顯示 |
| created_at | 建立時間 |
