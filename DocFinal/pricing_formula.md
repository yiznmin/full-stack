# 定價公式

> 狀態：基礎成本已確認，係數待定

> **定價公式 vs 實際成本公式的分野**
>
> 本文件所有公式用於**售價決定**（給客戶看的價格），採「色數 × NT$1」做簡化估算，便於客戶與管理員直觀理解。
>
> 實際顏料成本（用於庫存管理、進貨計算）採精算公式，見 `admin_color.md §2.8`：
> ```
> required_ml = max(畫布面積 × 該色佔比 × paint_ml_per_cm2 × paint_buffer_ratio, paint_min_ml)
> ```
>
> **兩者目的不同，不需要對齊**：
> - 定價公式：快速決定市場售價，規則簡單易懂
> - required_ml：精算每種顏色消耗多少 ml，用於庫存扣減、進貨判斷
>
> 若未來發現「色數 × NT$1」嚴重偏離真實成本，再考慮在定價公式中引入 required_ml 因子。

## 基礎成本計算方式

```
基礎成本 = 固定費 + 畫布印製費 + 框架費
（不含顏料，顏料成本納入難易度係數）
```

**固定費：NT$25**
| 項目 | 費用 |
|------|------|
| 畫具（筆刷等）| NT$5 |
| 包裝 | NT$20 |

**畫布印製費**
```
實際印製尺寸 = (畫布寬 + 10cm) × (畫布高 + 10cm)（四周各加 5cm 裝訂用）
印製費 = ceil(實際面積 ÷ 900) × NT$45
```
> 印刷廠最低起印 NT$200，但每次批量印刷皆超過此門檻，不影響單件成本。

**框架費**
```
框架費 = 畫布寬 + 畫布高（NT$）
邏輯：實木框以周長計費，單價 NT$0.5/cm
```

## 定價公式

```
售價 = (基礎成本 + 顏料費) × (1 + 細緻度加成率)
顏料費 = 色數 × NT$1
```

**難易度 → 色數**

顏料費依 `production_job.num_colors_used`（演算法實際使用色數）計算，非難易度預設上限。

| 難易度 | 色數上限（參考）| 顏料費計算 |
|------|------|------|
| 入門 | 18色 | num_colors_used × NT$1 |
| 初級 | 24色 | num_colors_used × NT$1 |
| 中級 | 35色 | num_colors_used × NT$1 |
| 進階 | 50色 | num_colors_used × NT$1 |

> 同難易度的不同商品因實際色數不同，售價可能略有差異。系統在建立變體時自動帶入公式計算值，管理員可手動調整。

**細緻度加成率**

| 細緻度 | 加成率 | 倍數 |
|------|------|------|
| 粗糙 | 60% | ×1.60 |
| 標準 | 75% | ×1.75 |
| 細緻 | 80% | ×1.80 |
| 高級 | 85% | ×1.85 |

> 錨點：30×40 入門 粗糙 = (NT$230 + NT$18) × 1.60 = **NT$397**

## 客製服務報價

### 客製照片（custom_photo）

**報價公式（與目錄商品同結構，外加客製服務費率）**

```
客製售價 = (基礎成本 + 顏料費) × (1 + 細緻度加成率) × 客製服務費率
```

- **客製服務費率**：預設 **2.0 倍**，存於 `system_settings.custom_photo_price_multiplier`，管理員可後台調整
- **顏料費**：依 `production_job.num_colors_used`（實際色數）× NT$1 計算
- **細緻度**：由管理員根據照片品質決定，不需客戶指定
- **加費項目**（`custom_photo_surcharges`）：表結構保留但初期**全部停用**（`is_active=false`）；未來視營運需求再啟用特定項目

**範例（30×40 入門 標準，實際色數 20）**

```
基礎成本 = NT$230
顏料費 = 20 × NT$1 = NT$20
細緻度加成 = 1.75（標準）
客製服務費率 = 2.0
公式價格 = (230 + 20) × 1.75 × 2.0 = NT$875
```

> 對照目錄同規格售價 NT$434（色數 18 時），客製約為目錄的 2 倍，差異主要來自色數與服務費率的複合作用。

**後台報價流程**

1. 系統跑完 production_job 後自動帶入**建議報價**(公式計算)
2. **建議報價僅顯示於管理員介面當參考,不會自動送出給客戶**
3. 管理員可勾選啟用中的加費項目(初期全部停用,無項目可選)
4. 管理員可依實際情況(照片複雜度、客戶議價等)手動調整最終 `quoted_price`
5. 管理員點「送出報價 + 初稿」後,寄 email 給客戶,客戶看到的**永遠是管理員確認過的金額**

> **設計理由**:系統自動計算難以涵蓋所有業務情境(照片複雜度、客戶關係等),管理員最後審核是必要的品質控管。

**設計理由**
- 與目錄商品共用公式結構，易於維護與客戶理解
- 2.0 倍涵蓋客製的額外成本：個別溝通、SAM 遮罩、顏色對應無法重用、客戶最多 3 次修改
- 倍數存於 system_settings，未來可依市場回饋調整
- 加費項目保留彈性，未來可快速啟用針對特定類型的加費

### 客製規格（custom_spec）

```
客製規格售價 = 目錄公式價格（不加客製服務費率）
```

> **設計理由**：custom_spec 是針對既有目錄圖片的尺寸 / 難度 / 細緻度調整，模板已存在，額外成本僅為重跑一次 production_job；不同於 custom_photo 的全新模板製作，因此不收客製服務費。

管理員確認即可，可手動微調。

**報價明細範例**

| 項目 | 說明 | 金額 |
|------|------|------|
| 基礎成本 | 40×50 | NT$295 |
| 顏料費 | 中級 35色 × NT$1 | NT$35 |
| 細緻度加成 | 標準 ×1.75 | — |
| **公式價格** | (295+35) × 1.75 | **NT$578** |
| **最終 quoted_price** | | **NT$578** |
